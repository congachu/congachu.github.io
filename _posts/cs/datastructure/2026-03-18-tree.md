`트리(Tree)` 는 이름 그대로 나무를 닮은 자료구조이다.  
트리 ADT는 뿌리인 root 노드에서부터 가지처럼 뻗어나가는 구조를 가진다.

# 트리

## 트리의 개념

트리는 앞서 설명한 것처럼 나무를 닮은 자료구조이다.  
나무에는 뿌리가 있고, 뿌리에서 가지가 뻗어나오며, 가지 끝에는 잎이 달린다.

트리 구조는 컴퓨터 과학에서 매우 자주 사용된다.  
운영체제의 파일 시스템은 트리 구조로 이루어져 있고, HTML이나 XML 문서를 다룰 때 사용하는 DOM 역시 트리 구조를 가진다.  
또한 검색 엔진이나 데이터베이스도 트리 자료구조를 기반으로 구현되는 경우가 많다.

이번에 구현할 트리 구조는 다음과 같다.

- `LCRS Tree`
- `Binary Tree`

---

## LCRS Tree 구현

`LCRS Tree` 는 `Left-Child Right-Sibling Tree` 의 약자이다.  
왼쪽에는 자식 노드를 연결하고, 오른쪽에는 같은 레벨의 형제 노드를 연결하는 방식으로 트리를 표현한다.

이 구조의 장점은 **자식의 수가 정해지지 않은 일반 트리도 이진 연결 구조처럼 단순하게 표현할 수 있다는 점**이다.

### 노드 구현

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.leftChild = None
        self.rightSibling = None
```

LCRS 트리에 필요한 구성에 맞게 노드를 정의하였다.

### LCRS Tree 클래스 정의

```python
class LCRSTree:
    def __init__(self):
        self.root = None
```

### 루트 노드 생성

```python
def createRoot(self, node: Node):
    if self.root != None:
        print("이미 루트가 존재합니다.")
        return False
    self.root = node
    return True
```

루트 노드 생성과 자식 노드 삽입을 분리하였다.

### 자식 노드 삽입

```python
def addChild(self, parent: Node, child: Node):
    if parent.leftChild != None:
        self.addSibling(parent.leftChild, child)
    else:
        parent.leftChild = child
    return True
```

### 형제 노드 삽입

```python
def addSibling(self, sibling: Node, node: Node):
    cur_node = sibling
    while cur_node.rightSibling != None:
        cur_node = cur_node.rightSibling
    cur_node.rightSibling = node
    return True
```

### 트리 전위 출력

```python
def printTree(self):
    print("-" * 10)
    self.preorder(self.root)
    print("\n" + "-" * 10)
    return True

def preorder(self, node: Node):
    if node == None:
        return
    print(node.data, end=" ")
    if node.leftChild != None:
        self.preorder(node.leftChild)
    if node.rightSibling != None:
        self.preorder(node.rightSibling)
    return
```

### LCRS Tree 시간복잡도

| 연산 | 시간복잡도 | 설명 |
|---|---|---|
| `createRoot` | O(1) | 루트 존재 여부 확인 후 대입 |
| `addChild` | O(1) 또는 O(k) | 첫 자식이 없으면 O(1), 있으면 형제 끝까지 이동 |
| `addSibling` | O(k) | 형제 노드 개수만큼 순회 |
| `preorder` | O(n) | 모든 노드를 한 번씩 방문 |
| `printTree` | O(n) | 내부적으로 전위 순회 수행 |

여기서 `k`는 같은 부모를 가진 형제 노드 수이고, `n`은 전체 노드 수이다.

즉, 현재 구현에서는 자식 노드를 맨 뒤에 추가할 때 형제들을 끝까지 순회해야 하므로  
`addChild()` 가 항상 O(1)은 아니다.

### 전체 코드

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.leftChild = None
        self.rightSibling = None

class LCRSTree:
    def __init__(self):
        self.root = None

    def createRoot(self, node: Node):
        if self.root != None:
            print("이미 루트가 존재합니다.")
            return False
        self.root = node
        return True
    
    def addChild(self, parent: Node, child: Node):
        if parent.leftChild != None:
            self.addSibling(parent.leftChild, child)
        else:
            parent.leftChild = child
        return True
    
    def addSibling(self, sibling: Node, node: Node):
        cur_node = sibling
        while cur_node.rightSibling != None:
            cur_node = cur_node.rightSibling
        cur_node.rightSibling = node
        return True
    
    def printTree(self):
        print("-" * 10)
        self.preorder(self.root)
        print("\n" + "-" * 10)
        return True
    
    def preorder(self, node: Node):
        if node == None:
            return
        print(node.data, end=" ")
        if node.leftChild != None:
            self.preorder(node.leftChild)
        if node.rightSibling != None:
            self.preorder(node.rightSibling)
        return
            

if __name__ == "__main__":
    lcrsTree = LCRSTree()
    nodeA = Node("A")
    nodeB = Node("B")
    nodeC = Node("C")
    nodeD = Node("D")
    nodeE = Node("E")
    nodeF = Node("F")

    lcrsTree.createRoot(nodeA)
    lcrsTree.createRoot(nodeB)
    lcrsTree.addChild(nodeA, nodeB)
    lcrsTree.addSibling(nodeB, nodeC)
    lcrsTree.addSibling(nodeB, nodeD)
    lcrsTree.addSibling(nodeD, nodeE)
    lcrsTree.addChild(nodeB, nodeF)
    lcrsTree.printTree()
```

## 이진 트리 구현

이진 트리는 하나의 노드가 자식을 최대 2개까지만 가질 수 있는 구조이다.  
즉, 노드의 차수는 2를 넘을 수 없다.

### 노드 구현

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None
```

### 이진 트리 클래스 정의

```python
class BinaryTree:
    def __init__(self):
        self.root = None
```

### 루트 노드 생성

```python
def createRoot(self, node: Node):
    if self.root != None:
        print("루트가 이미 존재합니다.")
        return False
    self.root = node
    return True
```

### 왼쪽 자식 노드 삽입

```python
def addChildLeft(self, parent: Node, child: Node):
    if parent.left != None:
        print("이미 왼쪽 자식 노드가 존재합니다.")
        return False
    parent.left = child
    return True
```

### 오른쪽 자식 노드 삽입

```python
def addChildRight(self, parent: Node, child: Node):
    if parent.right != None:
        print("이미 오른쪽 자식 노드가 존재합니다.")
        return False
    parent.right = child
    return True
```

### 전위, 중위, 후위 트리 출력

```python
def printTree(self):
    if self.root == None:
        print("트리 비어있음")
        return False
    print("전위" + "-" * 10)
    self.preorder(self.root)
    print("\n중위" + "-" * 10)
    self.inorder(self.root)
    print("\n후위" + "-" * 10)
    self.postorder(self.root)
    print("\n" + "-" * 10)

def preorder(self, node: Node):
    if node == None:
        return
    print(node.data, end=" ")
    self.preorder(node.left)
    self.preorder(node.right)
    return

def inorder(self, node: Node):
    if node == None:
        return
    self.inorder(node.left)
    print(node.data, end=" ")
    self.inorder(node.right)
    return

def postorder(self, node: Node):
    if node == None:
        return
    self.postorder(node.left)
    self.postorder(node.right)
    print(node.data, end=" ")
    return
```

### 이진 트리 시간복잡도

| 연산 | 시간복잡도 | 설명 |
|---|---|---|
| `createRoot` | O(1) | 루트 존재 여부 확인 후 대입 |
| `addChildLeft` | O(1) | 왼쪽 자식 존재 여부 확인 후 연결 |
| `addChildRight` | O(1) | 오른쪽 자식 존재 여부 확인 후 연결 |
| `preorder` | O(n) | 모든 노드를 한 번씩 방문 |
| `inorder` | O(n) | 모든 노드를 한 번씩 방문 |
| `postorder` | O(n) | 모든 노드를 한 번씩 방문 |
| `printTree` | O(n) | 순회 3번 수행 → O(3n) = O(n) |

여기서 `n`은 전체 노드 수이다.

### 전체 코드

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

class BinaryTree:
    def __init__(self):
        self.root = None

    def createRoot(self, node: Node):
        if self.root != None:
            print("루트가 이미 존재합니다.")
            return False
        self.root = node
        return True
    
    def addChildLeft(self, parent: Node, child: Node):
        if parent.left != None:
            print("이미 왼쪽 자식 노드가 존재합니다.")
            return False
        parent.left = child
        return True

    def addChildRight(self, parent: Node, child: Node):
        if parent.right != None:
            print("이미 오른쪽 자식 노드가 존재합니다.")
            return False
        parent.right = child
        return True
    
    def printTree(self):
        if self.root == None:
            print("트리 비어있음")
            return False
        print("전위" + "-" * 10)
        self.preorder(self.root)
        print("\n중위" + "-" * 10)
        self.inorder(self.root)
        print("\n후위" + "-" * 10)
        self.postorder(self.root)
        print("\n" + "-" * 10)

    def preorder(self, node: Node):
        if node == None:
            return
        print(node.data, end=" ")
        self.preorder(node.left)
        self.preorder(node.right)
        return

    def inorder(self, node: Node):
        if node == None:
            return
        self.inorder(node.left)
        print(node.data, end=" ")
        self.inorder(node.right)
        return

    def postorder(self, node: Node):
        if node == None:
            return
        self.postorder(node.left)
        self.postorder(node.right)
        print(node.data, end=" ")
        return

if __name__ == "__main__":
    bTree = BinaryTree()
    nodeA = Node("A")
    nodeB = Node("B")
    nodeC = Node("C")
    nodeD = Node("D")
    nodeE = Node("E")
    nodeF = Node("F")
    nodeG = Node("G")

    bTree.createRoot(nodeA)
    bTree.addChildLeft(nodeA, nodeB)
    bTree.addChildLeft(nodeB, nodeC)
    bTree.addChildRight(nodeB, nodeD)
    bTree.addChildRight(nodeA, nodeE)
    bTree.addChildLeft(nodeE, nodeF)
    bTree.addChildRight(nodeE, nodeG)
    bTree.printTree()
```

## 트리 구현 시간복잡도 정리

| 자료구조 | 연산 | 시간복잡도 |
|---|---|---|
| LCRS Tree | 루트 생성 | O(1) |
| LCRS Tree | 자식 추가 | O(1) 또는 O(k) |
| LCRS Tree | 형제 추가 | O(k) |
| LCRS Tree | 전위 순회 | O(n) |
| Binary Tree | 루트 생성 | O(1) |
| Binary Tree | 왼쪽 자식 추가 | O(1) |
| Binary Tree | 오른쪽 자식 추가 | O(1) |
| Binary Tree | 전위 순회 | O(n) |
| Binary Tree | 중위 순회 | O(n) |
| Binary Tree | 후위 순회 | O(n) |

## 참고할 점

일반 트리와 이진 트리에서 **특정 값을 찾는 탐색 연산**은 보통 최악의 경우 모든 노드를 방문해야 하므로 `O(n)` 이다.

즉, 트리는 무조건 빠른 탐색을 보장하는 구조가 아니다.  
탐색을 빠르게 만들려면 이후 배우게 되는 **이진 탐색 트리(BST)**, **AVL 트리**, **레드-블랙 트리** 같은 구조가 필요하다.
