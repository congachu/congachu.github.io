# 스택

## 스택의 개념

`스택(Stack)` 은 건초나 짚더미처럼 무언가를 차곡차곡 쌓아 올린 구조를 뜻한다.  
스택 ADT 역시 바닥에서부터 데이터를 쌓아 올리는 형태로 동작한다.

스택은 **후입선출(LIFO, Last In First Out)** 또는 **선입후출(FILO, First In Last Out)** 구조를 가진다.  
즉, 가장 마지막에 들어간 데이터가 가장 먼저 나오고, 먼저 들어간 데이터는 가장 나중에 나온다.

스택은 자동 메모리, 네트워크 프로토콜, 컴파일러의 구문 분석기, 되돌리기 기능 등 다양한 곳에서 활용된다.

스택을 구현하는 방법은 크게 두 가지가 있다.

- 배열 기반 스택
- 링크드 리스트 기반 스택

배열 기반 스택과 링크드 리스트 기반 스택의 장단점은 각각 배열과 링크드 리스트의 장단점과 유사하다.

---

## 스택 ADT

스택이 제공해야 하는 기본 기능은 다음과 같다.

- 탐색 → `peek`
- 삽입 → `push`
- 삭제 → `pop`
- 스택 상태 확인 → `isEmpty`, `isFull`

## 배열 기반 스택 구현

### 배열 구현

```python
class Array:
    def __init__(self, size):
        self.data = [None] * size

    def get(self, index):
        if self.checkOutOfRange(index):
            return False
        return self.data[index]
    
    def set(self, index, value):
        if self.checkOutOfRange(index):
            return False
        self.data[index] = value
        return True

    def checkOutOfRange(self, index):
        return self.len() <= index
    
    def len(self):
        return len(self.data)
```

파이썬에는 C나 Java처럼 고정 길이 배열이 따로 존재하지 않기 때문에, 여기서는 배열처럼 동작하는 클래스를 직접 구현하였다.

### 배열 기반 스택 클래스 정의

```python
class ArrayStack(StackADT):
    def __init__(self, size):
        self.top = -1
        self.data = Array(size)
```

- `top` 은 현재 스택의 가장 위에 있는 요소의 인덱스를 가리킨다.
- 스택이 비어 있으면 `top = -1` 이다.

### 삽입 함수

```python
def push(self, data):
    if self.isFull():
        print("배열이 가득 찼습니다.")
        return False

    self.top += 1
    self.data.set(self.top, data)
    return True
```

### 삭제 함수

```python
def pop(self):
    if self.isEmpty():
        print("배열이 비어있습니다.")
        return None

    result = self.data.get(self.top)
    self.data.set(self.top, None)
    self.top -= 1
    return result
```

### 탐색 함수

```python
def peek(self):
    if self.isEmpty():
        print("배열이 비어있습니다.")
        return None

    return self.data.get(self.top)
```

### 상태 확인 함수

```python
def isEmpty(self):
    if self.top == -1:
        return True
    return False

def isFull(self):
    if self.data.len() == (self.top + 1):
        return True
    return False
```

### 배열 기반 스택 시간복잡도

| 연산 | 시간복잡도 | 설명 |
|---|---|---|
| `push` | O(1) | top 증가 후 삽입 |
| `pop` | O(1) | top 위치 데이터 제거 |
| `peek` | O(1) | top 위치 데이터 확인 |
| `isEmpty` | O(1) | top 값 비교 |
| `isFull` | O(1) | 배열 길이와 top 비교 |

배열 기반 스택은 `top` 인덱스를 이용해 가장 위 요소에 바로 접근할 수 있으므로 대부분의 연산이 O(1)이다.

### 전체 코드

```python
import os
import sys

sys.path.append(os.path.abspath(os.path.dirname(os.path.abspath(os.path.dirname(__file__)))))

from adts import StackADT

class Array:
    def __init__(self, size):
        self.data = [None] * size

    def get(self, index):
        if self.checkOutOfRange(index):
            raise IndexError()
        return self.data[index]
    
    def set(self, index, value):
        if self.checkOutOfRange(index):
            raise IndexError()
        self.data[index] = value
        return True

    def checkOutOfRange(self, index):
        return self.len() <= index or 0 > index
    
    def len(self):
        return len(self.data)
    
class ArrayStack(StackADT):
    def __init__(self, size):
        self.top = -1
        self.data = Array(size)

    def push(self, data):
        if self.isFull():
            print("배열이 가득 찼습니다.")
            return False
        self.top += 1
        self.data.set(self.top, data)
        return True
    
    def pop(self):
        if self.isEmpty():
            print("배열이 비어있습니다.")
            return None
        result = self.data.get(self.top)
        self.data.set(self.top, None)
        self.top -= 1
        return result
    
    def peek(self):
        if self.isEmpty():
            print("배열이 비어있습니다.")
            return None
        return self.data.get(self.top)
    
    def isEmpty(self):
        if self.top == -1:
            return True
        return False
    
    def isFull(self):
        if self.data.len() == (self.top + 1):
            return True
        return False
    
    def printStack(self):
        if self.isEmpty():
            print("배열이 비어있습니다.")
            return False
        print("-" * 10)
        for i in range(self.top, -1, -1):
            print(f"| {self.data.get(i)} |")
        print("-" * 10)
        return True

if __name__ == "__main__":
    aStack = ArrayStack(5)
    print(aStack.pop())
    aStack.push(1)
    aStack.push(2)
    aStack.push(3)
    aStack.printStack()
    aStack.push(4)
    aStack.push(5)
    aStack.push(6)
    aStack.printStack()
    print(aStack.peek())
    print(aStack.pop())
    print(aStack.pop())
    print(aStack.peek())
    print(aStack.pop())
    print(aStack.pop())
    print(aStack.pop())
    print(aStack.pop())
```

## 링크드 리스트 기반 스택 구현

### 링크드 리스트 구현

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList(ListADT):
    def __init__(self):
        self.head = None
        self.size = 0
        self.tail = None

    def append(self, data):
        new_node = Node(data)
        if self.head == None:
            self.head = new_node
            self.tail = new_node
        else:
            self.tail.next = new_node
            self.tail = new_node
        self.size += 1
        return True

    def remove(self, data):
        prev = None
        if self.head == None:
            return False
        cur_node = self.head
        while cur_node != None and cur_node.data != data:
            prev = cur_node
            cur_node = cur_node.next
        
        if cur_node == None:
            return False
        if cur_node == self.head:
            if self.size == 1:
                self.__init__()
                return True
            else:
                self.head = self.head.next
                cur_node.next = None
        elif cur_node == self.tail:
            self.tail = prev
            self.tail.next = None
        else:
            prev.next = cur_node.next
        
        self.size -= 1
        return True
    
    def insert(self, data, index):
        if self.size <= 0:
            self.append(data)
            return True
        elif index <= 0:
            new_node = Node(data)
            new_node.next = self.head
            self.head = new_node
        elif index >= self.size:
            self.append(data)
            return True
        else:
            cur_node = self.head
            for _ in range(index - 1):
                cur_node = cur_node.next
            new_node = Node(data)
            new_node.next = cur_node.next
            cur_node.next = new_node
        self.size += 1
        return True

    def search(self, data):
        cur_node = self.head
        while cur_node != None:
            if cur_node.data == data:
                return True
            cur_node = cur_node.next
        return False

    def print(self):
        if self.size == 0:
            return False
        cur_node = self.head
        cur_num = 0
        print("[노드 번호 | 데이터]")
        while cur_node:
            print(f"[ {cur_num} | {cur_node.data} ]")
            cur_node = cur_node.next
            cur_num += 1

    def prepend(self, data):
        new_node = Node(data)
        if self.size == 0:
            self.tail = new_node
        else:
            new_node.next = self.head
        self.head = new_node
        self.size += 1
        return True
    
    def popFront(self):
        if self.size == 0:
            return None
        result = self.head.data
        if self.head == self.tail:
            self.head = self.tail = None
        else:
            temp = self.head
            self.head = self.head.next
            temp.next = None
        self.size -= 1
        return result
    
    def peekHead(self):
        if self.size == 0:
            return None
        return self.head.data
```

기존 링크드 리스트를 확장하여 스택에서 사용할 수 있는 연산을 추가하였다.

- `prepend()` : 맨 앞에 삽입
- `popFront()` : 맨 앞 요소 제거
- `peekHead()` : 맨 앞 요소 확인

스택은 가장 위에서만 삽입과 삭제가 일어나므로, 링크드 리스트의 `head`를 스택의 top처럼 사용하면 효율적으로 구현할 수 있다.

### 링크드 리스트 기반 스택 클래스 정의

```python
class LinkedListStack(StackADT):
    def __init__(self):
        self.data = LinkedList()
```

### 삽입 함수

```python
def push(self, data):
    return self.data.prepend(data)
```

### 삭제 함수

```python
def pop(self):
    return self.data.popFront()
```

### 탐색 함수

```python
def peek(self):
    return self.data.peekHead()
```

### 상태 확인 함수

```python
def isEmpty(self):
    return self.data.size == 0
```

링크드 리스트 기반 스택은 고정 크기 제한이 없기 때문에 일반적으로 `isFull()` 은 구현하지 않는다.

### 링크드 리스트 기반 스택 시간복잡도

| 연산 | 시간복잡도 | 설명 |
|---|---|---|
| `push` | O(1) | head 앞에 바로 삽입 |
| `pop` | O(1) | head 바로 제거 |
| `peek` | O(1) | head 데이터 확인 |
| `isEmpty` | O(1) | size 비교 |

링크드 리스트 기반 스택은 삽입과 삭제를 리스트 맨 앞에서 수행하므로 모든 핵심 연산이 O(1)이다.

### 전체 코드

```python
import os
import sys

sys.path.append(os.path.abspath(os.path.dirname(os.path.abspath(os.path.dirname(__file__)))))

from adts import StackADT
from List.LinkedList import LinkedList

class LinkedListStack(StackADT):
    def __init__(self):
        self.data = LinkedList()

    def push(self, data):
        return self.data.prepend(data)
    
    def pop(self):
        return self.data.popFront()
    
    def peek(self):
        return self.data.peekHead()
    
    def isEmpty(self):
        return self.data.size == 0
    
    def printStack(self):
        if self.isEmpty():
            print("스택이 비어있습니다.")
            return False
        cur_node = self.data.head
        print("-" * 10)
        for i in range(self.data.size - 1, -1, -1):
            print(f"| {i}번째 | {cur_node.data} |")
            cur_node = cur_node.next

if __name__ == "__main__":
    lls = LinkedListStack()
    print(lls.push(1))
    print(lls.push(2))
    print(lls.push(3))
    lls.printStack()
    print(lls.peek())
    print(lls.pop())
    print(lls.pop())
    lls.printStack()
    print(lls.pop())
    print(lls.pop())
```

## 배열 기반 스택과 링크드 리스트 기반 스택 비교

| 항목 | 배열 기반 스택 | 링크드 리스트 기반 스택 |
|---|---|---|
| `push` | O(1) | O(1) |
| `pop` | O(1) | O(1) |
| `peek` | O(1) | O(1) |
| 크기 제한 | 있음 | 없음 |
| 메모리 사용 | 고정 크기 | 필요한 만큼 동적 사용 |
| 구현 난이도 | 비교적 쉬움 | 상대적으로 복잡 |

배열 기반 스택은 구현이 단순하고 빠르지만 크기가 고정된다는 단점이 있다.  
반면 링크드 리스트 기반 스택은 크기 제한이 없지만, 노드와 포인터를 관리해야 하므로 구현이 조금 더 복잡하다.
